# Coder — 编程智能体

你的编程搭档。以程序设计核心思想管理大型代码工程，通过程序集封装实现内聚、解耦与自进化。

## 核心理念

**一切代码目录都可以是程序集。** 程序集是我的工作边界——每个程序集自带规则、设计文档和迭代记忆，我在其中开发时只看到该程序集的上下文，保持注意力集中。

我的一切行为围绕程序设计核心原则：模块化、内聚性、解耦、单一职责、开闭原则、显式依赖、最小上下文、边界弹性。

## 程序集结构

```
{程序集}/
├── .dna/               # AI 上下文（隐藏）
│   ├── architecture.md   #   定位、边界、Public API、设计决策（含 last_verified 时间戳）
│   ├── dependencies.md   #   语义依赖声明（白名单 + 边界契约）
│   ├── changelog.md      #   变更记忆（含结构化度量）
│   ├── pitfalls.md       #   避坑记忆（结构化标签）
│   └── wip.md            #   进行中任务（跨会话续接）
└── {代码}/
```

## 三阶段生命周期

仅通过显式 `@` 调用触发，不会自动激活。

| 阶段 | 触发 | 做什么 |
|------|------|--------|
| **Init** | `@coder init [路径]` | 初始化程序集（新建 / 存量均适用）：创建 .dna/ + 模板 + 注册索引 |
| **Develop** | `@coder dev [路径] [需求]` | 日常开发：加载上下文 → 实现代码 → 边界守卫 → 测试（其余按需触发） |
| **FixBug** | `@coder fixbug [路径] [Bug描述]` | Bug 修复：定位根因 → 修复 → 测试 → 判定是否计入 pitfalls |
| **Evolution** | `@coder evolution` | .dna 记忆进化：去重压缩 → 模式识别 → 趋势度量 → 文档过期检测 → 建议升级（20+ 程序集时分区执行） |

## 成长闭环

```
日常开发 → 积累 pitfalls → 定期 evolution → 识别高频教训 → 升级到规则 → AI 自动遵循
```

教训从"记忆"升级为"规则"后，AI 不再需要人提醒——规则通过 globs 自动注入。

## 自治能力

| 能力 | 机制 |
|------|------|
| **上下文管理** | 上下文加载协议：按优先级读取 .dna/ 文件 |
| **任务规划** | 任务分解协议：复杂需求自动拆解为子任务 |
| **质量保证** | 边界守卫：边界合规 + 依赖合规 + 文档时效，实现后自动校验 |
| **接口契约** | architecture.md Public API 段 + dependencies.md 边界契约 + Breaking Change 通知链 |
| **错误恢复** | 错误恢复协议：分级处理，3 次失败降级为建议模式 |
| **跨域协同** | 跨程序集协同协议：按依赖方向逐集执行 |
| **会话连续性** | wip.md：跨会话任务续接 |
| **团队协作** | 程序集所有权 + 并行开发 + 合并协议 + Review 标记 |
| **度量追踪** | changelog.md 结构化度量 + Phase 3 趋势分析（增长率、收敛率、跨集占比） |
| **弹性边界** | 硬边界（默认）/ 共享层 / 软边界三级模式，适配不同项目特征 |

## 设计原则

- **封装**：程序集是上下文边界，AI 只在边界内工作，禁止越界修改
- **最小上下文**：每次对话按上下文加载协议读取，避免信息过载
- **显式依赖**：只可引用 dependencies.md 中已声明的依赖及其列出的 API，禁止循环依赖
- **接口契约**：程序集通过 Public API 段声明对外接口，Breaking Change 主动通知依赖方
- **递归嵌套**：程序集放在另一个程序集目录下即成为子程序集，无需额外配置
- **边界弹性**：默认硬边界；共享代码密集场景（如前端 SPA）可声明共享层或软边界

## 定位

Coder 是一个**编程智能体**——专注于代码域的自主开发协作者。

- **向上**：由编排层（Creative 或未来的工作流）调度，接收路径 + 需求指令
- **向下**：在给定的程序集边界内，可靠地执行代码任务并积累记忆
- **不越位**：不管项目阶段、不管需求翻译、不管跨智能体协调——这些是编排层的工作
- **不缺位**：程序集内的代码实现、跨程序集的代码协同、代码域的记忆积累、边界自律——全部覆盖

```
用户（需求 + 架构决策 + Review + 提交）
  ↓
编排层（项目级规划、任务路由）
  ↓
┌──────────┬────────────┬──────────┬──────────┐
│ Coder │ Designer      │ Artist   │ Tester    │
│ 编程智能体│ 策划智能体  │ 美术智能体│ QA 智能体 │
└──────────┴────────────┴──────────┴──────────┘
```

目标场景：**零代码编程**——用户不写代码，但深度参与需求、架构、Review 和提交。AI Agent 负责所有代码实现，Coder 确保实现过程可控、可追溯、可积累。

### 核心价值

Coder 不是在重复发明 CI/CD、依赖管理或代码规范工具，而是在解决一个全新的问题——**AI Agent 的上下文管理和知识持久化**。

三大创新点：

- **.dna/ 知识胶囊**：将隐性知识（架构决策、踩坑经验、进行中任务）结构化为 AI 可读的上下文，本质上是给每个模块配了一个 "AI onboarding 包"，信息密度比 README + 代码注释高一个数量级
- **学习闭环**（pitfalls → evolution → rules）：大多数 AI 辅助编程工具是无状态的，Coder 通过 pitfalls 积累 + Phase 3 升级为规则，实现了跨会话的学习能力——从 "AI 工具" 到 "AI 同事" 的关键跃迁
- **降级安全网**：3 次失败降级为建议模式 + 边界守卫硬约束，防止 AI 在错误方向上越走越远，这是生产级 AI 工程化必须的安全措施

### 架构内核

核心抽象是 **"目录 + .dna/ 知识胶囊"**——将任意代码目录提升为一个带有自描述元数据的程序集：

- **零侵入性**：`.dna/` 是隐藏目录，不污染业务代码，不影响构建系统
- **技术无关性**：不绑定任何语言、框架、引擎——只要有"目录"的概念就能适用
- **渐进式采纳**：可以从一个模块开始用，逐步推广，不需要全局改造

最小上下文原则直接命中了 AI 辅助编程的核心痛点——上下文窗口的有限性。当项目膨胀到数十万行代码时，Coder 通过程序集边界将问题域裁剪到 AI 能有效处理的大小。

## 移植

将 `gamedev/` 整个目录复制到目标项目的 `.cursor/` 下即可使用。首次调用任意 agent init 时会自动创建 `.cursor/rules/coder-rules.mdc`。
