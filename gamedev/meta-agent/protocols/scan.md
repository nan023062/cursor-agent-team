# 扫描协议

> `@meta scan` 时执行。扫描工程中已有的智能体和 .dna/ 目录，评估存在性与内容健康度。

## 步骤

### 1. 扫描智能体

读取 `../`（智能体所在目录），找出所有含 `AGENT.md` 或 `SKILL.md`（旧版兼容）的子目录。

对每个智能体，读取定义文件元数据（name / description），提取：
- 智能体名称
- 一句话定位
- 管辖范围（globs）
- 是否有对应的 `.cursor/rules/{agent}-rules.mdc`
- 是否有 Evolution 段（即是否有进化能力）

### 2. 扫描 .dna/ 目录

递归扫描工程根目录，找出所有含 `.dna/` 子目录的路径。排除常见构建目录（`node_modules` `.git` `Library` `Temp` `Logs` `obj` `bin`）。

对每个 `.dna/`，统计：
- 文件数量和列表
- 是否有 architecture 文件（L3 规则）
- 是否有 pitfalls 文件（L1 经验）
- architecture 的 maturity 值（如有）

### 3. 内容健康度检查

对每个 `.dna/` 中的文件进行深度检查：

#### architecture 健康度

| 检查项 | 判定 | 阈值 |
|--------|------|------|
| 文件存在但「概述」「核心模型」「边界」段均为空或仅含模板占位符 | 🔲 空壳 | 有效内容 < 5 行 |
| `last_verified` 超过 30 天未更新 | ⚠️ 过期 | 30 天 |
| `last_verified` 超过 90 天未更新 | 🔴 严重过期 | 90 天 |
| `last_verified` 字段不存在 | ⚠️ 缺字段 | — |
| `maturity` = 0 且文件创建超过 30 天 | 💤 休眠 | 创建 30 天后仍未升格 |
| 「约束」段为空 | ℹ️ 待积累 | — |

**空壳检测方法**：读取文件内容，剔除注释行（`<!-- -->`）、模板占位符（`{== ... ==}`、`{MODULE_TERM}` 等未替换变量）和空行后，计算剩余有效行数。

#### pitfalls 健康度

| 检查项 | 判定 | 阈值 |
|--------|------|------|
| 文件存在但无任何条目（仅有模板头部） | 🔲 空壳 | 条目数 = 0 |
| 最近一条记录超过 60 天 | 💤 休眠 | 60 天 |
| 条目数 ≥ 20 且从未 Evolution | ⚠️ 待清理 | 20 条 + maturity = 0 |
| 同一标签出现 ≥ 3 次但未升格 | 🔔 可升格 | 同标签 ≥ 3 |

**条目计数方法**：匹配以 `- [` 开头的行（标准条目格式）。

#### changelog 健康度

| 检查项 | 判定 | 阈值 |
|--------|------|------|
| 文件存在但仅含初始化条目 | 🔲 空壳 | 条目数 ≤ 1 且内容为模板默认 |
| 最近一条记录超过 60 天 | 💤 休眠 | 60 天 |

### 4. 扫描智能体自身 .dna/

对每个智能体目录，检查是否存在 `.dna/pitfalls.md`（智能体流程摩擦记录）。

| 检查项 | 判定 | 说明 |
|--------|------|------|
| `.dna/pitfalls.md` 不存在 | ❌ 缺失 | 智能体无法积累流程摩擦经验 |
| 存在但无条目 | ℹ️ 空 | 正常（新智能体或流程顺畅） |
| 同一标签 ≥ 3 次 | 🔔 可改进 | 建议改进 AGENT.md 对应流程/规则/模板 |
| 条目数 ≥ 10 且从未处理 | ⚠️ 待处理 | 积压过多，建议逐条审查并改进智能体 |

**标签统计**：按 `#FlowBlock` `#RuleMissing` `#TemplatePoor` `#CrossAgentGap` 分别计数，输出频率表。

### 5. 匹配

将 `.dna/` 目录与智能体的管辖范围（globs）匹配，标记：
- 属于哪个智能体管辖
- 未被任何智能体管辖的孤立 .dna/

### 6. 快捷指令一致性检查

扫描 `.cursor/commands` 下带 `<!-- [meta-commands] -->` 或 `<!-- [brain-commands] -->` 标记的文件，与 Step 1 提取的智能体指令清单比对：

| 情况 | 判定 | 说明 |
|------|------|------|
| 智能体有指令但无对应 command 文件 | ❌ 缺失 | 建议执行 `@meta sync` |
| command 文件存在但智能体已删除该指令 | ⚠️ 孤立 | 建议删除 |
| command 文件内容与当前智能体指令不一致 | ⚠️ 漂移 | 建议执行 `@meta sync` 更新 |
| 匹配一致 | ✅ 同步 | — |
| `.cursor/commands` 目录不存在 | ℹ️ 未初始化 | 建议执行 `@meta sync` 生成 |
